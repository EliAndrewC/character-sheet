{# Shared "Copy as image" button. Every roll-result panel renders this so the player can drop a PNG of the result on the clipboard for pasting into Discord. Bindings: * ``rollImageStatus`` - 'idle' | 'rendering' | 'ready' | 'copied' | 'failed' Drives the label text + colour + disabled state, and the ``x-show`` guard that hides the button entirely when no prerender has been kicked off for this panel. * ``rollImageBlob`` - the resolved PNG that ``copyRollImage()`` writes to ``navigator.clipboard``. * ``copyRollImage()`` - the click handler. The button is the same shape and styling across every result panel so a player who learns it in one modal recognises it in the rest. Each calling panel is responsible for calling ``_prerenderRollImage()`` at the right phase transition and resetting ``rollImageStatus = 'idle'`` when a new roll for that modal begins. #} {# Desktop: copy the PNG to the clipboard for pasting into Discord. Hidden on touch devices that can use the OS share sheet (``_useShareImage``), where the clipboard path is a dead end (Discord mobile pastes the clipboard URI as text, not the image). #} {# Mobile: hand the PNG to the OS share sheet so it can go straight into Discord. Shown in place of Copy on share-capable touch devices. #}
